~Id3Copy

Описание:

Предназначена для работы с тегами ID3 популярного формата MP3. С ее помощью можно копировать теги.
Работа ведется только с тегами ID3v1, то есть не поддерживаются длинные комментарии, жанры и расширенная информация о песне.
  1. Входной файл. Справа - кнопка для выбора. В поле входных файлов можно использовать маски (такие же, как при обычном поиске в операционной системе. Например, *.mp3.
  2. Выходной файл. Если имя выходного файла совпадает с именем входного файла, то тег просто модифицируется. Также можно использовать маски:
    • * или *.* - заменяется на реальное имя обрабатываемого файла с расширением
    • *.ext - заменяется на имя файла с расширением .ext
    • name.* - заменяется на файл name с прежним расширением
    • {filename} - заменяется на полное имя файла
    • {filename i1} - заменяется на кусок имени файла, начиная с i1 позиции
    • {filename i1 i2} - заменяется на кусок имени файла с i1 позиции длины i2
    • {name}, {name i1}, {name i1 i2} - то же, но только для имени файла (без расширения)
    • {ext}, {ext i1}, {ext i1 i2} - то же самое, но для расширения (с точкой)
  3. Глубина сканирования. Самая левая позиция - без сканирования подкаталогов, самая правая - самый глубокий режим сканирования. Выберите, что вам нужно.
  4. Нужен ли тег в выходном файле. Если снято, то тег в выходном файле стирается. В этом случае логично было бы указать исходный и конечный файл одинаковыми (см. пункт 2)
  5. Содержимое обобщенного тега. Здесь в особом виде записано то, что будет записано в ID3 тег в выходной файл. В полях допустимы следующие константы:
    • {Filename} - имя файла
    • {Name} - имя файла (без пути и расширения)
    • {Ext} - расширение файла (с предваряющей точкой)
    • {Title} - заголовок, название
    • {Artist} - исполнитель
    • {Album} - альбом
    • {Comment} - комментарий
    • {Year} - год
    • {Genre} - жанр
    После константы через пробел можно записать два целых числа в десятичной нотации. Тогда будет вставлен кусок константы в указанном интервале (первое число - начало, второе - длина куска).
    Пример: {Filename 1 10} - отрезать от имени файла первые 10 символов
    Либо можно указать только первый символ - тогда скопируется остаток до конца строки:
    {FileName 5} - взять с пятого символа имени файла до конца.
  6. Пресеты. То есть можно заранее создать варианты переделок тегов, а затем только выбирать нужный. По умолчанию в файле конфигурации присутствует пресет Копировать - перенос в выходной файл тега входного без изменений. Кнопки:
    • + - добавить пресет. Программа запросит вас его имя.
    • - - удалить выделенный пресет.
    • * - модифицировать выделенный пресет. Следует запомнить, что сперва нужно выбрать нужный пресет, затем модифицировать теги и лишь потом его модифицировать. Программа запросит имя пресета (если нужно, его можно поменять).
  7. Кнопка запуска преобразования. Текущий преобразуемый файл будет отображаться в статусной строке под кнопками. Также запуск преобразования можно осуществить по кнопке Enter в полях редактирования.
  8. Выход с сохранением параметров. Также выйти из программы можно по нажатию Esc.

Особенности:

  • Пустой жанр имеет код 255! Это особенность формата MP3
