Параметры командной строки

Запуск:

TypeAndRun.exe [--settings=path] [--config=path] [--configold=path] [--history=path] [--plugins=path] [--add=path] [--exec=string]

Где:

--settings=path - указать файл настроек. Если параметр --settings не указан, то используется файл TypeAndRun.ini в главной папке программы. Path - полный путь к *.ini файлу. Если в Path указано только имя файла, то используется главная папка программы. Параметр игнорируется, если TypeAndRun уже запущен.

--config=path - указать файл конфига с алиасами. Если параметр --config не указан, то используется файл Config.ini в главной папке программы. Path - полный путь к *.ini файлу. Если в Path указано только имя файла, то используется главная папка программы. Параметр игнорируется, если TypeAndRun уже запущен.

--configold=path - указать файл конфига с удаленными мертвыми алиасами. Если параметр --configold не указан, то используется файл, находящийся там же и имеющей такое же имя (с добавлением '~'), что и файл конфига. Path - полный путь к *.ini файлу. Если в Path указано только имя файла, то используется главная папка программы. Параметр игнорируется, если TypeAndRun уже запущен.

--history=path - указать файл истории. Если параметр --history не указан, то используется файл History.ini в главной папке программы. Path - полный путь к *.ini файлу. Если в Path указано только имя файла, то используется главная папка программы. Параметр игнорируется, если TypeAndRun уже запущен.

--plugins=path - указать файл настроек плагинов. Если параметр --plugins не указан, то используется файл Plugins.ini в главной папке программы. Path - полный путь к *.ini файлу. Если в Path указано только имя файла, то используется главная папка программы. Параметр игнорируется, если TypeAndRun уже запущен.

--add=path - добавить алиас в конфиг. Если параметр --add не указан, то добавление не происходит. Path - полный путь к файлу или папке. Если Path опущен, то добавление не происходит. Параметр работает, даже если TypeAndRun уже запущен.

--exec=string - запустить строку string с помощью консоли. Команда аналогична набору строки в консоли и нажатия Enter.

--multiusers=[1\0] - указывает программе, в каком режиме работать с конфигами. Если указать 0, то будут использоваться конфиги из директории TypeAndRun. Если 1, то используются конфиги из директории %APPDATA%\TypeAndRun. Если же параметр --multiusers не указан, то используются настройки TypeAndRun из реестра.

В том случае, когда в параметре path или string встречаются пробелы, его необходимо взять в кавычки.

Порядок следования параметров может быть произвольным.

Внимание! Неправильное указание вышеприведенных параметров может привести к ошибке и потере ваших конфигов! Сохраните все файлы *.ini перед тем, как экспериментировать с параметрами командной строки.
