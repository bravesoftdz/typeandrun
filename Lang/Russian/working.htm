Настройка программы

Appearance

Show console window on start - Показывать ли окно консоли при запуске программы или запускать в свернутом режиме.
Hide console with Enter - Скрывать ли окно консоли после запуска с нее строки или оставлять на экране.
Trigger show (Hotkey or Esc) - Скрывать ли окно консоли при повторном нажатии горячей клавиши или делать это по нажатию Escape.
Show icon in system tray - Отображать ли иконку в системном трее.
Show balloon tips in system tray (ME and >=NT5 only) - Показывать всплывающие сообщения в системном трее.
Put mouse cursor into console every show console window - Помещать курсор мышки в окно консоли при каждом ее вызове. Полезно при одновременном использовании X-Mouse в системе и автоскрытия консоли.
AutoHide console after (in millisecs) - Автоматически скрывать окно консоли в случае потери фокуса через настраиваемый промежуток времени.
Running priority - Приоритет, с которым запускается консоль. Рекомендуется - высокий.

Console look

Show in center X - Показывать окно консоли в центре экрана по координатам X или самому их указать.
Show in center Y - Показывать окно консоли в центре экрана по координатам Y или самому их указать.
Console width - Ширина окна консоли в пикселах.
(in percent of screen width) - Указывать ширину окна консоли не в пикселях, а в процентах от ширины экрана.
Console height - Высота окна консоли в пикселах.
Border size - Размер рамки около окна консоли в пикселах.
Show border only (OS >= NT5) - Не отображать поле для ввода (работает только на системах NT > 5 версии).
Transparent console - Прозрачность окна консоли, от 0 до 255, причем 0 - полная прозрачность (работает только на системах NT > 5 версии).

Hint popup

Show hint while typing alises and system aliases in console - Показывать подсказку по набираемым алиасам и системным алиасам. Подсказка состоит из строки, которая будет отдана на выполнение по нажатию Enter, а также из комментария к алиасу, если он был сделан в конфиге.
Border size - Размер рамки около окна подсказки в пикселах.
Show border only (OS >= NT5) - Не отображать поле для ввода (работает только на системах NT > 5 версии).
Transparent console - Прозрачность окна подсказки, от 0 до 255, причем 0 - полная прозрачность (работает только на системах NT > 5 версии).

Dropdown list

Number of lines in the dropdown list - Максимальное число линий в выпадающем списке дополнений.
Border size - Размер рамки около окна выпадающего списка в пикселах.
Show border only (OS >= NT5) - Не отображать поле для ввода (работает только на системах NT > 5 версии).
Transparent console - Прозрачность окна выпадающего списка, от 0 до 255, причем 0 - полная прозрачность (работает только на системах NT > 5 версии).

Colors

Из выпадающего списка надо выбрать, для какого режима работы консоли редактировать цвета:
EasyType disable - Для выключенного автодополнения.
EasyType enable - Для включенного автодополнения.
BASH model - Для дополнения как в BASH.
Text color - Цвет шрифта консоли.
Console color - Цвет консоли (фон шрифта).
Border color - Цвет рамки около окна консоли.
Hint text color - Цвет шрифта подсказки.
Hint color - Цвет фона подсказки.
Hint border color - Цвет рамки около окна подсказки.
Dropdown text color - Цвет шрифта выпадающего списка.
Dropdown color - Цвет фона выпадающего списка.
Dropdown border color - Цвет рамки около окна выпадающего списка.

Font

Change font - Смена шрифта в консоли. Сохраняются сведения о типе шрифта, его размере и начертания.
Autosize font to console height - Автоматически подгонять размер шрифта под высоту окна консоли.
Text alignment - Можно выбрать, как выравнивается текст в консоли:
Left align - По левому краю.
Center align - По центру.
Right align - По правому краю.
Apply English layout every hotkey - Включать английскую раскладку при каждом вызове консоли. Полезно, если все алиасы на английском, но русский может иногда использоваться в качестве параметра.
Lock English layout in the console - Запретить изменять раскладку клавиатуры в консоли. Полезно, если русский вообще не будет использоваться в консоли.

HotKey

Смена горячей клавиши, по которой показывается окно консоли. Настраивается на совесть пользователя, так как некоторые комбинации просто невозможны.

EasyType

Don't use EasyType - Не использовать автодополнение.
Use EasyType - Использовать стандартное автодополнение. Дополненная часть выделяется, что позволяет легко ее изменить.
Use BASH model of EasyType - Использовать автодополнение на клавише TAB - как в linux. Выделение не используется.
Use EasyType for internal commands - Автодополнение из списка внутренних команд.
Use EasyType for aliases - Автодополнение из названий алиасов.
Use EasyType for system aliases - Автодополнение из названий системных алиасов.
Use EasyType for path browser - Автодополнение путей - тот самый весь пробаженный браузер путей.
Use EasyType for history - Автодополнение из истории.
Add space to alias name - При автодополнении алиаса добавлять в конец пробел.
Case sensitivity (www <> WWW) - Различать регистр набираемых символов или нет.
Enable EasyType every console push - Включать автодополнение при каждом вызове консоли.
Use advanced BackSpace push - Включить продвинутый режим обработки клавиши BackSpace. Если в консоли есть выделенный текст, то при нажатии этой клавиши удалится не только он, но и один символ слева от выделенного текст. И в завершение снова включится дополнение (если оно разрешено в настройках). Таким образом избегаются двойные нажатия BackSpace при неправильном наборе алиаса.
Delete duplicates in the EasyType completion list - Позволяет автоматически удалять одинаковые строки из списка автодополнения, но может замедлять работу на огромных списках.

Exec

Execute internal commands - Запускать с консоли внутренние команды.
Execute aliases - Запускать с консоли алиасы.
... with default shell action - Если в поле action у алиаса прописан документ, то обычно для его открытия используется метод оболочки "open". Эта настройка позволяет запускать документы с действитем по умолчанию (как двойной щелчёк на файле), но может вызвать проблемы с запуском в Windows 98 и NT4.
Execute system aliases - Запускать с консоли системные алиасы.
Execute URL's - Запускать с консоли адреса интернета.
Execute E-Mail's - Запускать с консоли адреса почты.
Execute folders - Запускать с консоли папки.
Execute files - Запускать с консоли файлы.
Execute shell commands (all other) - Запускать с консоли все, что не подходит к вышеперечисленному, через шелл.

History

Don't save history to file - Вообще не сохранять историю в файл. История, что уже была сохранена, не удаляется.
Save history to file - Сохранять историю в файл. Ниже каждый случай подробнее настраивается.
Save internal commands - Сохранять внутренние команды. Смысла нет, ибо они и так зашиты в программе.
Save internal commands parameters - Сохранять внутренние команды с переданными им параметрами.
Save alias - Сохранять названия алиасов. Смысла нет, ибо они и так хранятся в конфиге.
Save alias parameters - Сохранять названия алиасов с переданными им параметрами.
Save system alias - Сохранять названия системных алиасов. Смысла нет, ибо они и так хранятся в реестре.
Save system alias parameters - Сохранять названия системных алиасов с переданными им параметрами.
Save URL's - Сохранять адреса интернета.
Save E-Mail's - Сохранять адреса почты.
Save folder's - Сохранять пути папок. Смысла нет, ибо папки и так валяются на харде.
Save shell commands (all other AKA unknown) - Сохранять все остальное, запускаемое с консоли. Будут сохраняться и опечатки.
Save strings from command line (--exec=string) - Сохранять строки, переданные через командную строку (параметр --exec=). Причем сохраняться будет в соответствии с типом запускаемой строки и настроек выше.
Save strings from message WM_COPYDATA to console window - Сохранять строки, переданные через сообщение WM_COPYDATA окно консоли TfrmTARConsole. Причем сохраняться будет в соответствии с типом запускаемой строки и настроек выше.
Invert history scroll - Обычно клавиша или колесико мыши вверх прокручивают историю назад. Эта настройка позволяет изменить направление прокрутки.
Max history strings (0 - infinite) - Максимальное количество строк в истории. В случае переполнения происходит сдвиг - стирается самая старая запись и добавляется новая в начало.
Clear history - Очистить всю историю. Стирается сразу по нажатию кнопки.

Paths

Your internet browser - Интернет-браузер, через который запускаются адреса интернета.
Your file manager (example: explorer.exe) - Файловый менеджер, через который запускаются папки.
Your command shell (example: command.com) - Шелл, через который запускаются неклассифицированные строки. Стандартные: для 9x - command.com, для NT - cmd.exe.
Parameter for command shell - Enter: - Параметр, передаваемый шеллу, с помощью которого запускается неизвестная строка с консоли при нажатии Enter.
Parameter for command shell - Shift+Enter: - Параметр, передаваемый шеллу, с помощью которого запускается неизвестная строка с консоли при нажатии Shift+Enter.
Запуск происходит так: %shell% %shell_param% %console_string%

Registry

Данные параметры строятся на основе значений реестра. Если пользователь не имеет прав для изменения данного значения реестра, то параметр будет недоступен.
No autostart - Не запускать программу вместе с операционной системой.
HKLM - for all users - Запускать программу через ключ реестра HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\run\TypeAndRun. TypeAndRun будет запускаться для всех пользователей.
HKCU - for current user - Запускать программу через ключ реестра HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\run\TypeAndRun. TypeAndRun будет запускаться только для текущего пользователя.
Shell integration ("Add to TypeAndRun..." context menu) - Добавить в контекстное меню файлов и папок меню, с помощью которого можно будет легко их вносить в программу.
Use multi users config system (NT only) - Позволяет каждому пользователю системы использовать свои собственные конфиги. Хранятся они в папке %APPDATA%\TypeAndRun. В однопользовательском режиме используются конфиги из директории программы. Параметр доступен только для NT

Language

Select language - Выбрать нужный язык из списка. Отображаются автор перевода и версия программы, к которой он создавался.

Undo

Use Undo function - Использовать ли функцию отмены в консоли. Полезно, если набираешь что-то неимоверно огромное в консоли, но тут запускается какая-нибудь наглая программа и перехватывает фокус. А консоль настроена на автоскрытие. Тогда жмешь хоткей, а потом Ctrl+Z - все набранное восстанавливается. Больше смысла в применении данной фичи я не вижу - не текстовой же это редактор!

Sounds

Enable sounds - Включить звуки в программе.
Execute internal command - Проигрывать звук при запуске внутренней команды.
Execute alias - Проигрывать звук при запуске алиаса.
Execute system alias - Проигрывать звук при запуске системного алиаса.
Execute URL - Проигрывать звук при запуске ссылки интернета.
Execute E-Mail - Проигрывать звук при запуске адреса электронной почты.
Execute folder - Проигрывать звук при запуске папки.
Execute file - Проигрывать звук при запуске файла.
Execute plugin - Проигрывать звук при запуске плагина.
Execute all other - Проигрывать звук при запуске всех остальных команд.
В поле внизу можно вручную отредактировать путь к звуковому файлу или по нажатию Browse выбрать файл с помощью диалога.

Config

Move executed alias to first position of config - Перемещать запущенный алиас в самый вверх конфига. В этом случае конфиг конфигурируется таким образом, что самые часто запускаемые алиасы оказываются вверху для облегчения автодополнения.
Autosort config by alias name - Автоматически сортировать конфиг по имени алиаса в возрастающем порядке.
Auto reread Config.ini every (in millisecs) - Автоматически перечитывать конфиг с алиасами и системные алиасы через заданный промежуток времени. Полезно для тех, кто редактирует конфиг вручную, в смысле - не средствами данной программы.
Reread Config.ini every show console window - Перечитывать конфиг с алиасами и системные алиасы при каждом нажатии горячей клавиши, т.е. при каждом вызове консоли.
Create ~Config.ini files with removed dead aliases? - Создавать файл ~Config.ini с удаленными из основного конфига мертвыми алиасами.
Get confirmation before removing dead alias - Делать запрос на применение изменений пути и удаление алиаса.
Search file on all drives but another folder - Искать файл с тем же названием, что указано в действии алиаса, на всех дисках компьютера. Полезно, если файл изменил местоположение, например та же программа установлена в другую папку, или в случае разницы наименований дисков в разных системах (CED в 98 и CDE в XP).
Check dead aliases on every execute TypeAndRun - Делать проверку на мертвые алиасы каждый раз при запуске TypeAndRun.
Не рекомендуется совмещать последние две настройки на слабых компьютерах или на компьютерах с большим количеством дисков, тем более сетевых.
Check dead aliases - Проверить конфиг на мертвые алиасы. Внимание - все изменения сохраняются немедленно!
Get back all dead aliases - Вернуть удаленные мертвые алиасы в конфиг, если они были сохранены.

Edit Config

Здесь происходит редактирование алиасов. У каждого алиаса обязательны два параметры - alias name и action. Параметры у алиаса можно редактировать отдельно или же в том виде, в каком он сохранен в конфиге.
+ - Применяет изменения, сделанные в поле ввода полного представления алиаса.
New Alias - Создает новый алиас.
Delete Alias - Удаляет выделенный алиас.
Горячие клавиши в листе алиасов:
Insert - Создать новый алиас.
Delete - Удалить текущий алиас.
Ctrl+UpArrow - Переместить текущий алиас на позицию вверх.
Ctrl+DownArrow - Переместить текущий алиас на позицию вниз.
Ctrl+Home - Переместить текущий алиас в самый верх конфига.
Ctrl+End - Переместить текущий алиас в самый низ конфига.
Алфавитно-цифровые клавиши - Быстрый переход на нужный алиас, начинающийся с данных символов. После этого стрелочки UpArrow или DownArrow - переход по алиасам, начинающимся с набранной последовательности. Возврат обратно в список алиасов - Escape.
Перемещать алиасы по конфигу можно также перетаскиванием мыши или с помощью кнопок-стрелочек.
P - Позволяет выбрать в качестве действия алиаса строку из плагина. Кнопка неактивна, если ни один плагин не содержит готовых алиасов.

Plugins

Здесь перечислены подключенные плагины. По каждому плагину выводится информация: имя плагина, его версия, краткое описание, местонахождение и прочее.
Load plugin - Загружает незагруженный уже подключенный плагин.
Unload plugin - Выгружает загруженный уже подключенный плагин.
Add plugin - Позволяет подключить новый плагин к программе. В окошке выбора *.dll файла можно выбрать сразу несколько плагинов.
Delete> plugin - Выгружает и удаляет выделенный плагин из списка.
У каждого плагина есть свойства. Изменять их можно только когда плагин подключен и загружен.
Load plugin on TypeAndRun start - Определяет, будет ли загружаться плагин автоматически при старте программы. Тем не менее, его можно загрузить\выгрузить и вручную при помощи внутренних команд /load и /unload.
Save history for this plugin - Позволяет определить, надо ли сохранять историю команд для этого плагина. Например, для tar_math сохранение истории удобно выключить.
Если у выбранного плагина есть плагинные алиасы, то показывается список со всеми доступными плагинными алиасами. Двойной клик по одному из плагинных алиасов в этом списке позволяет сделать пробный запуск.
Перемещать плагины по списку с тем, чтобы более важные плагины для пользователя обрабатывались первыми, можно с помощью кнопок-стрелочек.
