Q1: I cannot execute commands like 'ping'
A1: Open the Settings dialog and tick "Execute shell commands (all other)" checkbox. Please also check the command shell settings. For Windows 95, 98, Me usually command.com, for Windows NT4, 2000, XP - cmd.exe.
 
Q2: I have created an alias in the configuration file, but it does not work
A2: There can be several reasons for that:
1) Alias will not work if there is '|' symbol in it.
2) You have not updated the configuration file in the program memory. Exit TypeAndRun and start it again, or click the Reread Config option in the context menu, or enter the console and press Ctrl+R, or type '/reread'.
3) Check if there are any white spaces in alias, folder path or parameter. Try to put double quotes around the alias string in the config.ini file.
4) Bug? Mail me!
 
Q3: How can I see a web page? I enter astalavista.box.sk, and get nothing!
A3: If the address you type does not start with "www.", then you have to type "http://" as well. Also, you have tick the "Execute URL's" checkbox in the settings.

Q6: The program says "Can't bind hotkey! Now use system tray icon..."?

Q7: My History file is full of crap! How can I clean it?
A7: You have the History.ini file... Simply erase it. Or click the Clear History button in the Settings Dialog. If you do not want to use the History any more, just click Don't Save History to File radio button in the Settings Dialog or check out checkboxes you do not want to use.
 
Q8: Can Type&Run exec shortcuts? When this feature will be available in Type&Run?
A8: Type&Run can execute shortcuts, you need to write path to the shortcut in "action" field of the alias. If shortcut is still not executes then you need to add some keys to you registry:
============ файл *.reg === начало ===================================
REGEDIT4

[HKEY_CLASSES_ROOT\lnkfile]
"IsShortcut"=""

[HKEY_CLASSES_ROOT\piffile]
"IsShortcut"=""
============ файл *.reg === конец ====================================

Q16: What are "system aliases"?
A16: System aliases are one of Windows "hidden" features. A lot of applications (and actually Windows itself, that's why we call them "system") during installation automatically create subkeys in "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\" Registry key with such a structure:
Key name - this is an alias name - in most cases linked executable filename (e.g. "Winword.exe").
String values inside a key:
"(Default)" - full path to executable (e.g. "C:\PROGRA~1\Office\OFFICE11\WINWORD.EXE);
"Path" - full path to executable location folder (e.g. "C:\PROGRA~1\Office\OFFICE11\), nonabligatory one.
So in most cases you already have that predefined aliases working the same way as defined in TypeAndRun configuration ones (TypeAndRun "understand" only linked to *.exe files system aliases) and you can save your time and affords using them.
