How to create aliases

1) Aliases are saved into the config.ini file in as follows:

alias|action[|[parameters][|[folder][|[state][|[history][|[to the top][|[priority][|comment]]]]]]]

alias:
Name assigned to the entry with all its parameters. You will have to type it in the console window to start specified application. It is a mandatory field.

action:
Any action allowed in the system. For example, it can be a full path to an application or a document, internet or e-mail address, as well as a shell command. It is also a mandatory field.

parameters:
Parameters automatically passed to the action. The final list of parameters will be compiled as follows: parameter_saved_in_the_config.ini_file + parameter_passed_through_the_console_window. There is no white space automatically added between the parameters to expand functionality of the program. If you need one, take care of it yourself. If you have to change the order of parameters, see how you can do it in the section 2.
"¦" (Alt+0166) symbol substitutes for "|" (pipe) in alias "Parameters:" field (you can't use "|" directly in alias configuration 'cause it's already reserved by TAR as field separator).

folder:
Current directory for the program. If no value is supplied, it will be substituted with the full path to the folder where the executed file is stored, when applicable.

state:
Initial state of the program after start:
0 - Hidden
1 - Standard (default value)
2 - Minimized
3 - Maximized
4 - Standard without focus
5 - Hidden without focus
Please note that each application may have its own settings, which may override your choice.

history:
How the parameters passed with the alias are saved:
0 - Same as in the general settings (default value)
1 - Always (if the alias has a parameter)
2 - Never

to the top:
Settings relating to position of the alias in the alias list after it is executed:
0 - Same as in the general settings (default value)
1 - Always
2 - Never

priority:
Priority which will be assigned to the program associated with the alias when it is started:
-1 - Low
0 - Normal (default value)
1 - High
2 - Real time
This option is ignored if the alias is associated with a document, folder or other non-executable files.

comment:
Small description of the alias, that for you to remember waht is alias purpose. Displays in hint while you type the alias name.

2) Replacing passed parameters

When specifying the application path or parameter or start path, you may add %s as a parameter. In this case, %s will be replaced with the parameter you pass to the program from the console window. For example, you may have the following alias for Google search:
gg|gg|http://www.google.com/search?q=%s&hl=en
Then the "gg test" (with no quotes) typed in the console will open a browser window with the following address:
"http://www.google.com/search?q=test&hl=en" (with no quotes)

When specifying the application path or parameter or start path, you may add several parameters: %1, %2 .. , %N . In this case, %N will be replaced with the N-count parameter you passed to the program from the console window. Space is delimeter for the parameters. For example:

alias|D:\Path\%1|name=%1 twoparam=%2 allparams=%s
Then the "alias 1.exe 2 3 4 5" (with no quotes) typed in the console, the
"D:\Path\1.exe name=1.exe twoparam=2 allparams=1.exe 2 3 4 5" (with no quotes)
will be executed.

3) Attach several commands to one alias

If you want to start several applications with a single alias, set up all these applications separately with the same alias name. Then you will be able to start them all typing just that one alias. And if you want to have more complicated functionality, it would be better to write a .bat or .cmd file and assign an alias to that.

4) Symbols not allowed in aliases

You should not use pipe symbol ("|"), since it is used to separate different parts of an alias in the config.ini file. Neither should you use white spaces in aliases, for there will be a problem to distinguish between an alias and its parameters. I would not recommend to use aliases with slash "/" as a first symbol, since it is used in the internal program commands.
